## 규칙
- 만들고 바로 main(default)브랜치에 푸시 절대 안됨!!!!
- Pull Request 하기, 요구사항 구체적으로 적어놓기!
- 커밋 메세지 규칙를 준수하며 커밋해주세요
- 각각 기능들은 Issue만들어서 브랜치 생성 후 작성해주세요.
  (다 만들어지면 pr 해주세요)  
  <br><br>
  |종류|내용|
  | ------ | ----------|
  | ADD   | 새로운 기능    |
  | FIX      | 버그 수정        |
  | REFACTOR | 코드 리팩토링    |
  | TEST     | 테스트 코드 작성 |
  | CHORE    | 자잘한 일        |
  | DOCS     | 문서 추가        |
  | DELETE   | 삭제             |

  예시
  ```
  commit -m "add) 사이드바 추가"
  ```
